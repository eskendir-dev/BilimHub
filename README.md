# BilimHub - Интеллектуальная экосистема Aqbobek Lyceum

---

## Требования для запуска

- Установленный **Node.js** (рекомендуется LTS).
- Установленный **npm** (обычно идет вместе с Node.js).

Проверка в терминале:
```bash
node -v
npm -v
```

---

## Быстрый старт (очень подробно)

### 1) Скачать проект
Если через Git:
```bash
git clone https://github.com/eskendir-dev/BilimHub.git
cd BilimHub
```

Если без Git:
- Нажмите **Code → Download ZIP**
- Распакуйте архив
- Откройте папку проекта в терминале

### 2) Установить зависимости
```bash
npm install
```

### 3) Запустить проект
```bash
npm run dev
```

После запуска терминал покажет адрес вроде:
`http://localhost:5173/`  
Открой его в браузере.

### 4) Остановить проект
В терминале нажмите:
`Ctrl + C`

## О проекте простыми словами

**BilimHub** — это веб-портал для школы, где:
- ученики смотрят оценки, кружки, расписание и получают рекомендации;
- родители видят данные своего ребёнка;
- учителя работают с оценками и кружками;
- администрация публикует новости, управляет расписанием и статистикой;
- киоск показывает публичную ленту/расписание для школьного монитора.

Проект работает **без реального сервера**.  
Все данные хранятся локально в браузере (через `localStorage`) и в mock-данных.

## Что внутри (фичи)

### 1) Авторизация и регистрация
- Вход по почте и паролю.
- Регистрация с выбором роли:
  - Ученик
  - Родитель
  - Учитель
  - Администрация
  - Киоск
- Для родителя при регистрации указывается почта ученика.

### 2) Лента новостей
- Общий раздел для всех ролей.
- Можно ставить “сердце” (лайк) на пост.
- Администрация может:
  - публиковать посты;
  - редактировать посты;
  - удалять посты.

### 3) Оценки
- Для ученика/родителя:
  - таблица вида `Предмет | КР | Сор1 | Сор2 | Соч`;
  - значения отображаются как `балл/макс_балл` (например, `15/20`).
- Для учителя:
  - выбор класса;
  - редактирование оценок прямо в таблице.

### 4) Кружки
- Ученик:
  - подписка/отписка на кружки.
- Родитель:
  - только просмотр кружков ребёнка.
- Учитель:
  - видит свои кружки;
  - может создать заявку на новый кружок.
- Администратор:
  - подтверждает/отклоняет заявки кружков.

### 5) Расписание
- Администратор может:
  - загрузить ссылку;
  - или прикрепить файл.
- Ученик/родитель/учитель/киоск видят расписание только в рамках своей роли:
  - ученик — своего класса;
  - родитель — класса ребёнка;
  - учитель — своих классов;
  - киоск — всех классов.

### 6) Чат
- Доступен всем, кроме:
  - родителя,
  - киоска.
- Ограничения:
  - обычные пользователи не пишут администрации;
  - администрация пишет только учителям.
- Есть “Недавние чаты” — после первого сообщения диалог сохраняется.

### 7) Профиль
- Просмотр имени, роли, класса, почты.
- Редактирование:
  - имени;
  - аватарки (ссылка или файл).
- Выход из аккаунта.

### 8) AI-логика (рекомендации)
- На основании оценок показывается рекомендация ученику
  (например, какой предмет подтянуть).

### 9) Kiosk Mode
- Режим для школьного экрана.
- Публичный вид ленты и расписания.

## Как хранится “база данных”

Проект использует:
1. **mockServer.js** — генерация/выдача мок-данных;
2. **localStorage** — “самосохраняемое” состояние в браузере.

Это значит:
- изменения (посты, профиль, чаты и т.д.) сохраняются после перезагрузки страницы;
- но только на вашем текущем браузере/устройстве.

---

## Сброс локальной базы (если что-то “сломалось”)

В консоли браузера (F12 → Console):
```js
localStorage.removeItem('bilimhub_full_hackathon_v4')
location.reload()
```

## Частые ошибки и решение

### Ошибка: `Failed to resolve import`
Проверьте правильность путей:
- `import App from './App';`
- `import './App.css';`

### Ошибка в `main.jsx`
Файл должен завершаться вызовом:
```jsx
ReactDOM.createRoot(document.getElementById('root')).render(...)
```

### Порт занят
Если `5173` занят, Vite предложит другой (например `5174`).
